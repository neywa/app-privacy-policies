ShiftFeed

Privacy Policy

Effective date: 22 April 2026  ·  Last updated: 22 April 2026

1 Introduction

ShiftFeed is a read-only news aggregator for the OpenShift and Kubernetes community, developed by an independent developer. It surfaces publicly available articles, GitHub release announcements, Red Hat security advisories, and an AI-generated daily briefing in a single feed.

The short version: ShiftFeed collects no personal data. There are no accounts, no analytics, no advertising, and no tracking. The only network activity is fetching public article content and, on mobile, delivering broadcast push notifications.

By using the app you agree to the practices described in this policy.

2 Data the App Does Not Collect

ShiftFeed does not collect, store, or transmit any of the following:

  • Name or email address
  • Location data
  • Device or advertising IDs
  • Usage analytics
  • Crash reports
  • Contacts or microphone
  • Camera or photos
  • Browsing history
  • Any biometric data
  • User-generated content

There are no user accounts, no sign-up flow, and no profile of any kind. No data about you is ever sent to any server operated by the developer.

3 Data the App Does Handle

Article Content (read-only)

The app fetches publicly available article metadata — title, URL, source name, summary, tags, and publication date — along with AI-generated daily briefings from a Supabase-hosted database. This content contains no user data; it is identical for every user who opens the app.

Push Notifications (Android and iOS only)

On Android and iOS, the app requests notification permission at first launch and subscribes the device to three broadcast topics via Firebase Cloud Messaging (FCM):

  • all — daily AI briefing
  • security — CVE and security advisory alerts
  • releases — new software release alerts

All notifications are broadcast to every subscribed device. None are personalised or targeted at an individual user. The Firebase SDK generates an FCM registration token that is managed by Google; the app does not transmit this token to any server operated by the developer. Push notifications are not available on the web or desktop versions of the app.

In-App Browser

On Android and iOS, tapping an article opens it in an embedded WebView. On web and desktop, it opens in the user's external browser. When you visit an article, the destination website (e.g. Red Hat Blog, Kubernetes Blog, CNCF Blog, Hacker News, GitHub) may set its own cookies and log the visit according to its own privacy policy. ShiftFeed does not inject scripts, intercept content, or collect anything from these browsing sessions.

Local Preferences

A light/dark theme preference is stored on-device only using standard platform local storage. It is never transmitted anywhere.

4 Device Permissions

Android

  • INTERNET — required to fetch articles and briefings from the backend database.
  • POST_NOTIFICATIONS — requested at runtime on Android 13 and above to deliver push notifications. You can deny or revoke this permission at any time in system settings; the rest of the app continues to function normally.

iOS

  • Notifications — requested at runtime to deliver push notifications. You can deny or revoke this permission at any time in system settings.

Web and Desktop

No special permissions are requested. The web version runs as a static site and makes only anonymous read requests to the backend database.

5 Third-Party Services

ShiftFeed relies on the following third-party services. No other SDKs, analytics tools, or advertising networks are included.

Supabase (database backend)

Article metadata and AI briefings are stored in a Supabase-hosted PostgreSQL database running on AWS infrastructure. The app makes anonymous read-only requests using a public publishable key. No user identity or personal data is transmitted in these requests.

Supabase privacy policy: supabase.com/privacy

Google Firebase Cloud Messaging

Used exclusively to deliver broadcast push notifications on Android and iOS. Subject to Google's privacy practices. The developer does not receive or store FCM registration tokens.

Google privacy policy: policies.google.com/privacy

GitHub Pages

The web version of ShiftFeed is hosted as a static site on GitHub Pages. Visitors may be subject to GitHub's privacy practices for static site visitors.

GitHub privacy policy: GitHub Privacy Statement

Anthropic Claude (server-side only)

The daily briefing text is generated on the server side by Anthropic's Claude API, using only the public article summaries already stored in the database. No user data is sent to Anthropic. Users do not interact with Claude directly.

Anthropic privacy policy: anthropic.com/legal/privacy

6 International Data Transfers

ShiftFeed itself collects no personal data, so there is nothing to transfer on our behalf. However, the third-party services the app relies on — Supabase (AWS infrastructure), Google Firebase, GitHub, and Anthropic — may process requests in the United States or other countries. Please refer to each provider's privacy policy for details of their data transfer practices.

7 Data Retention

The app stores no personal data, so there is nothing to retain or delete on the developer's side.

  • You can revoke notification permission at any time in your device's system settings, which ends FCM topic subscriptions.
  • Uninstalling the app removes the local theme preference from your device and terminates all FCM subscriptions.

8 Your Rights

Because ShiftFeed collects no personal data, standard data-subject rights — access, rectification, portability, erasure — are largely not applicable to the developer. There is no data held on our end to provide, correct, or delete.

If you are located in the European Union, you have rights under the General Data Protection Regulation (GDPR). If you are a California resident, you have rights under the California Consumer Privacy Act (CCPA). In both cases, since we hold no personal data, your primary point of control is your device's system settings (for notification permissions) and the app stores (for app data removal on uninstall).

For any questions, please contact us at the address in Section 10.

9 Children's Privacy

ShiftFeed is aimed at software engineers, SREs, and DevOps professionals. It is not directed at children under the age of 13 (or under 16 in jurisdictions where that threshold applies) and does not knowingly collect any data from children.

10 Security

Because ShiftFeed transmits no personal data, there is no personal data at risk in transit or at rest on our servers. The app's only outgoing requests are anonymous reads of public article content.

We recommend keeping your device's operating system up to date to benefit from the latest platform-level security protections.
